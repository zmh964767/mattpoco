# ADR-0003: AI模型策略（本地 + 第三方混合）

## 状态

已接受

## 背景

零职达的核心功能是"根据JD优化生成匹配简历"，这需要AI能力。面临以下约束：
- 用户数据隐私要求高（简历含敏感信息）
- 用户不愿承担高额API费用
- 不同用户电脑配置差异大（有的能跑本地模型，有的不能）
- 需要支持离线使用

## 决策

采用**本地模型优先 + 第三方API可选**的混合策略。

### 本地模型（默认）

| 组件 | 说明 |
|------|------|
| 运行环境 | Ollama / LM Studio / llama.cpp |
| 推荐模型 | Llama 3.1 8B / Qwen 2.5 7B / ChatGLM3-6B |
| 适用任务 | 简历优化、匹配度评分、JD摘要 |
| 优点 | 完全离线、零API费用、数据不出本机 |
| 缺点 | 需要8GB+显存/内存，生成速度较慢 |

### 第三方API（可选配置）

| 供应商 | 模型 | 适用场景 |
|--------|------|----------|
| 阿里云 | 通义千问 | 中文简历优化效果好 |
| 百度 | 文心一言 | 国内接入便捷 |
| OpenAI | GPT-4o | 英文JD、国际化求职 |
| Anthropic | Claude 3.5 | 长文本理解能力强 |

### 模型切换方式

- **用户手动选择**：在设置界面选择"本地模型"或"第三方API"
- **非自动切换**：不根据任务复杂度自动切换，避免用户困惑
- **默认推荐**：首次使用引导用户选择，推荐本地模型作为默认

### 数据脱敏流程

发送给第三方API前，必须执行脱敏：

```
原始简历
  ├── 姓名: 张三        ──►  [姓名]
  ├── 公司: 阿里巴巴     ──►  [公司A]
  ├── 电话: 138****1234 ──►  [电话]
  ├── 邮箱: zhang@***.com ──► [邮箱]
  ├── 学校: 清华大学     ──►  [学校]
  ├── 项目: 双十一系统   ──►  [项目A]
  └── 自我评价: 主导过千万级项目 ──► 主导过[规模]项目
```

脱敏映射表本地加密存储（AES-256），AI返回后还原真实信息。

### 任务分级

| 任务类型 | 复杂度 | 推荐模型 | 说明 |
|----------|--------|----------|------|
| JD关键词提取 | 低 | 本地小模型 | 简单NLP任务 |
| 匹配度评分 | 中 | 本地模型 | 结构化推理 |
| 简历内容优化 | 高 | 本地/第三方 | 创意写作，长文本生成 |
| 整份简历生成 | 高 | 第三方API | 质量要求最高 |

### 模型下载管理

- **首次使用引导**：检测本地是否安装Ollama，未安装则提供下载链接和安装指南
- **模型下载UX**：提供下载进度条、预计剩余时间、断点续传、取消按钮
- **存储位置**：模型文件存储在用户数据目录（如 `%APPDATA%/zerojob/models`），避免占用系统盘空间
- **版本管理**：支持多模型共存，用户可切换不同模型

### API Key 安全存储

- **加密存储**：第三方API Key使用AES-256加密后存储在本地配置文件
- **密钥派生**：加密密钥从用户主密码或系统密钥链派生
- **明文禁止**：不以任何形式明文保存API Key
- **输入安全**：设置界面输入API Key时显示为密码框，支持显示/隐藏切换

## 备选方案

### 方案A：纯第三方API
- **拒绝原因**：用户需承担API费用（优化一份简历可能消耗￥0.5-2）；隐私风险（敏感信息上传第三方）；无法离线使用

### 方案B：纯本地模型
- **拒绝原因**：低配置电脑无法运行；模型效果不如GPT-4等顶级模型；首次下载模型体积大（4-8GB）

### 方案C：云端自建模型服务
- **拒绝原因**：需要服务器和GPU资源，成本高昂；与"本地优先"的产品定位冲突

## 后果

### 正面
- 用户零成本即可使用基础功能
- 隐私保护最佳（本地模型数据不出本机）
- 高级用户可通过配置API Key获得更好效果
- 渐进式体验：先用本地模型，不满意再配置API

### 负面
- 本地模型效果受硬件限制
- 需维护多供应商API适配层
- 数据脱敏增加代码复杂度（需确保脱敏/还原逻辑无Bug）
- 模型首次下载体积大，用户体验受影响
