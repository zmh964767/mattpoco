# ADR-0002: 开源策略（AGPL-3.0 + 闭源增值）

## 状态

已接受

## 背景

零职达最初计划采用订阅制收费（3天试用 + 月费/年费），但评估后发现：
- 支付集成需要营业执照和服务器维护，MVP阶段成本过高
- 作为个人/小团队项目，开源可以快速获取用户反馈和社区贡献
- 核心差异化（AI提示词、模板设计）需要保护，防止被直接复制

## 决策

采用**AGPL-3.0开源协议 + 闭源增值内容**的混合策略。

### 仓库划分

| 仓库 | 协议 | 内容 | 可见性 |
|------|------|------|--------|
| `zerojob` | AGPL-3.0 | 桌面端核心、插件、解析规则、数据库Schema | 公开 |
| `zerojob-pro` | 私有 | AI提示词模板、高级简历模板、品牌资源 | 私有 |

### 发布包构成

```
ZeroJob-Installer.exe
├── 开源核心（来自 zerojob 仓库）
│   ├── 桌面端应用
│   ├── 浏览器插件
│   └── 基础简历模板
└── 闭源增值（来自 zerojob-pro 仓库）
    ├── AI提示词模板
    ├── 高级简历模板
    └── 品牌资源
```

### 协议选择理由

选择 **AGPL-3.0** 而非 MIT/Apache：
- **防止SaaS漏洞**：网络服务也必须开源，防止云服务商直接部署赚钱
- **保护社区贡献**：衍生作品必须开源，确保改进回馈社区
- **专利保护**：明确授予专利使用权

## 备选方案

### 方案A：完全开源（MIT协议）
- **拒绝原因**：MIT过于宽松，大公司可以随意拿去改改就闭源商业化，无法保护核心利益

### 方案B：完全闭源免费
- **拒绝原因**：无法获得社区贡献（解析规则维护、Bug修复）；用户信任度低；不符合"帮助求职者"的初心

### 方案C：GPL-3.0（非AGPL）
- **拒绝原因**：GPL只保护软件分发，不保护网络服务。有人可以在服务器上部署零职达提供服务，无需开源

## 后果

### 正面
- 快速建立用户基础和社区生态
- 解析规则等维护工作可由社区分担
- 个人技术影响力提升
- 后续可通过企业定制、技术支持等方式变现

### 负面
- AGPL可能让部分企业用户望而却步（担心传染其现有代码）
- 需维护两个仓库，发布流程稍复杂
- 闭源内容（提示词模板）的更新需手动同步到发布包
