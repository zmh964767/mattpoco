# PRD: 零职达 (ZeroJob) - AI驱动的简历投递助手

## Problem Statement

求职者在投递简历时面临以下痛点：
- **简历不匹配**：同一份简历投给不同公司/岗位，没有针对性优化，导致通过率低下
- **信息分散**：需要在多个招聘平台（BOSS直聘、智联招聘、猎聘、前程无忧、拉勾网）反复填写相同信息
- **岗位筛选耗时**：每天手动浏览大量岗位，难以快速识别最适合自己的机会
- **隐私担忧**：担心个人简历信息被平台滥用或泄露
- **成本顾虑**：现有AI简历工具大多按次收费，求职者经济压力大

## Solution

零职达是一款**完全开源免费**的AI简历投递助手，采用**桌面端 + 浏览器插件**混合架构：

- **本地优先**：所有用户数据存储在本地SQLite数据库，无需登录，无需上传服务器
- **AI简历生成**：根据岗位JD智能优化简历内容，生成针对性强的PDF/DOCX简历
- **多平台聚合**：自动爬取五大主流招聘平台岗位，本地统一管理和评分
- **隐私保护**：调用第三方AI前自动脱敏敏感信息；支持本地开源模型，完全离线使用
- **半自动投递**：一键跳转到招聘平台投递页，用户确认后完成投递

## User Stories

### 核心功能

1. 作为求职者，我想要录入我的基本信息（姓名、电话、邮箱、求职意向），以便系统了解我的基本情况
2. 作为求职者，我想要录入我的教育经历（学校、专业、学历、时间），以便生成完整的简历
3. 作为求职者，我想要录入我的工作/实习经历（公司、职位、时间、职责描述），以便展示我的工作经验
4. 作为求职者，我想要录入我的项目经历（项目名称、技术栈、职责、成果），以便突出我的技术能力
5. 作为求职者，我想要录入我的校园经历（社团、竞赛、志愿活动），以便丰富我的简历内容
6. 作为求职者，我想要添加自定义字段（如政治面貌、民族、籍贯等），以便满足特定行业/企业的要求
7. 作为求职者，我想要上传我的证件照，以便嵌入到生成的简历中
8. 作为求职者，我想要创建多份基础简历（如"互联网版"、"国企版"），以便针对不同行业准备不同侧重点的简历

### 岗位管理

9. 作为求职者，我想要设置爬取条件（城市、关键词），以便系统只抓取我感兴趣的岗位
10. 作为求职者，我想要支持多城市、多关键词组合的批量爬取，以便扩大求职范围
11. 作为求职者，我想要系统自动定时爬取岗位（每天凌晨），以便每天打开就能看到新岗位
12. 作为求职者，我想要查看已爬取的岗位列表，以便浏览和筛选
13. 作为求职者，我想要看到每个岗位的匹配度评分，以便快速识别最适合的机会
14. 作为求职者，我想要看到匹配度的分项得分（技能、经验、学历、薪资、地域），以便了解匹配细节
15. 作为求职者，我想要自定义各维度的评分权重，以便根据个人偏好调整匹配优先级
16. 作为求职者，我想要看到简洁的匹配度说明（如"技能匹配度高，但经验稍欠缺"），以便快速判断
17. 作为求职者，我想要岗位数据7天后自动标记为过期，以便避免投递已下架的岗位

### AI简历生成

18. 作为求职者，我想要选择一份基础简历和一个目标岗位，以便生成针对性的匹配简历
19. 作为求职者，我想要AI根据JD优化我的简历内容（重新措辞、合理扩展），以便提高通过率
20. 作为求职者，我想要选择极简风格的简历模板（1-3套），以便生成简约专业的简历
21. 作为求职者，我想要生成PDF格式的简历，以便通用投递
22. 作为求职者，我想要生成DOCX格式的简历，以便后续手动编辑
23. 作为求职者，我想要在生成的简历中嵌入我的证件照，以便完善简历外观
24. 作为求职者，我想要查看生成的简历历史版本，以便回溯和对比
25. 作为求职者，我想要直接覆盖现有版本生成新简历，以便快速迭代优化

### 投递流程

26. 作为求职者，我想要从岗位列表选择多个岗位批量生成简历，以便提高投递效率
27. 作为求职者，我想要一键跳转到招聘平台的投递页面，以便快速完成投递
28. 作为求职者，我想要在跳转时系统自动带上生成的简历文件路径，以便手动上传附件
29. 作为求职者，我想要记录每份简历的投递历史（公司、岗位、时间），以便追踪投递进度
30. 作为求职者，我想要查看某份简历的所有投递记录，以便了解这份简历的使用情况

### AI模型配置

31. 作为求职者，我想要选择本地AI模型（Ollama/LM Studio/llama.cpp），以便免费离线使用
32. 作为求职者，我想要配置第三方AI API Key（通义千问/文心一言/GPT-4等），以便获得更好的优化效果
33. 作为求职者，我想要在配置API Key时看到数据脱敏说明，以便了解我的隐私如何被保护
34. 作为求职者，我想要根据任务复杂度自动选择模型（简单任务用本地模型，复杂任务用第三方API），以便平衡成本和效果

### 浏览器插件

35. 作为求职者，我想要在浏览招聘网站时自动识别JD内容，以便快速添加到桌面端
36. 作为求职者，我想要点击插件图标手动抓取当前页面的JD，以便处理未自动识别的情况
37. 作为求职者，我想要插件与桌面端自动通信，以便无缝同步数据

### 解析规则热更新

38. 作为求职者，我想要系统自动检查解析规则更新，以便招聘网站改版后仍能正常爬取
39. 作为求职者，我想要新规则解析失败时自动回滚到旧版本，以便保证系统稳定性

## Implementation Decisions

### 模块划分

#### 深度模块（核心、可独立测试）

1. **ResumeEngine（简历引擎）**
   - 职责：管理求职者信息、基础简历、匹配简历的CRUD
   - 接口：`createBaseResume()`, `generateTargetedResume()`, `listVersions()`, `getDeliveryHistory()`
   - 数据模型：求职者信息、教育经历、工作/实习经历、项目经历、校园经历、基础简历、匹配简历、投递记录

2. **JobCrawler（岗位爬虫）**
   - 职责：定时爬取各招聘平台岗位，解析结构化数据
   - 接口：`crawl(platform, keywords, cities)`, `getParser(platform)`, `updateRules()`, `markExpired()`
   - 实现：Playwright无头浏览器模拟真实用户访问，内置解析器 + 热更新机制

3. **MatchingEngine（匹配引擎）**
   - 职责：多维度评分计算
   - 接口：`calculateScore(resume, job)`, `setWeights(weights)`, `explainScore()`
   - 算法：规则引擎基础分 + AI综合评估，五维度（技能、经验、学历、薪资、地域）

4. **AIOptimizer（AI优化器）**
   - 职责：数据脱敏、调用AI模型、简历内容优化
   - 接口：`optimize(resume, jd)`, `sanitize(data)`, `restore(data)`, `extractKeywords(jd)`
   - 策略：本地模型优先（Ollama/LM Studio），第三方API可选（通义千问/文心一言/GPT-4）

5. **DocumentGenerator（文档生成器）**
   - 职责：PDF/DOCX简历生成、模板渲染、证件照嵌入
   - 接口：`generatePDF(resume, template)`, `generateDOCX(resume, template)`, `renderTemplate()`
   - 模板：1-3套极简风格，支持证件照嵌入

6. **ExtensionBridge（插件桥接）**
   - 职责：桌面端HTTP服务，处理插件请求
   - 接口：`startServer(port)`, `handleJDPush()`, `handleDeliveryRedirect()`
   - 通信：桌面端启动localhost HTTP服务，插件通过标准HTTP请求通信

#### 辅助模块

7. **ConfigManager（配置管理）**
   - 职责：管理用户配置（AI模型设置、爬取条件、评分权重、主题等）
   - 存储：本地JSON文件或SQLite配置表

8. **UpdateManager（更新管理）**
   - 职责：检查解析规则更新、下载新规则、失败回滚
   - 源：GitHub公开仓库

### 技术栈

- **桌面端**：Tauri（Rust后端 + Web前端）
- **浏览器插件**：Manifest V3
- **数据库**：SQLite（本地存储）
- **爬虫**：Playwright（无头浏览器）
- **AI本地运行**：Ollama / LM Studio / llama.cpp
- **文档生成**：Rust PDF库（如genpdf）+ DOCX库
- **通信**：桌面端localhost HTTP服务

### 数据流

```
求职者录入信息 ──► ResumeEngine ──► SQLite
                                      │
JobCrawler爬取岗位 ──► SQLite ──► MatchingEngine计算评分
                                      │
用户选择岗位 ──► AIOptimizer优化 ──► DocumentGenerator生成PDF/DOCX
                                      │
浏览器插件读取JD ──► ExtensionBridge ──► 桌面端处理
                                      │
用户确认投递 ──► 插件跳转招聘平台 ──► 用户手动上传简历
                                      │
投递完成 ──► ResumeEngine记录历史 ──► SQLite
```

### 数据脱敏策略

调用第三方AI API前，必须执行以下脱敏：
- 姓名 → [姓名]
- 公司名 → [公司A]、[公司B]
- 电话 → [电话]
- 邮箱 → [邮箱]
- 学校名 → [学校]
- 项目名 → [项目A]

脱敏映射表本地存储，AI返回后还原真实信息。

### 岗位爬取策略

- **频率**：每天凌晨自动执行
- **范围**：用户配置的多城市 + 多关键词组合
- **反爬**：随机延迟（2-5秒）、请求频率控制、浏览器指纹模拟、Cookie持久化
- **过期**：7天未更新的岗位自动标记为过期

### 开源策略

- **公共仓库**（`zerojob`）：AGPL-3.0协议，核心代码完全开源
- **私有仓库**（`zerojob-pro`）：AI提示词模板、高级简历模板、品牌资源
- **发布包**：开源核心 + 闭源增值内容，用户免费使用

## Testing Decisions

### 测试原则

- **只测外部行为，不测实现细节**：测试模块的输入输出，不关心内部算法
- **单元测试优先**：每个深度模块必须有独立的单元测试
- **集成测试覆盖关键路径**：爬虫+解析器、AI优化+文档生成等端到端流程

### 模块测试计划

| 模块 | 测试类型 | 测试重点 |
|------|----------|----------|
| ResumeEngine | 单元测试 | CRUD操作、版本控制、投递历史记录 |
| JobCrawler | 单元测试 + Mock | 解析器正确性、规则热更新、失败回滚 |
| MatchingEngine | 单元测试 | 评分算法正确性、权重调整、边界条件 |
| AIOptimizer | 单元测试 + Mock | 脱敏/还原正确性、AI调用适配层 |
| DocumentGenerator | 单元测试 + 快照测试 | PDF/DOCX输出结构、模板渲染正确性 |
| ExtensionBridge | 集成测试 | HTTP API正确性、插件通信流程 |

### 测试工具

- **Rust后端**：内置测试框架 `cargo test`
- **Web前端**：Vitest + Testing Library
- **浏览器插件**：Playwright E2E测试

## Out of Scope

以下功能不在本次PRD范围内，可作为后续迭代：

1. **全自动投递**：模拟登录招聘平台并自动提交简历（法律风险高、技术难度大）
2. **在线表单预填**：自动填写招聘平台的在线简历表单（浏览器安全限制）
3. **实时通知**：岗位更新推送、投递状态变更通知（需要服务器和推送服务）
4. **社交功能**：简历分享、求职交流社区（与产品核心定位不符）
5. **多语言支持**：英文简历生成、国际化招聘平台（专注国内主流平台）
6. **移动端App**：iOS/Android原生应用（桌面端+插件已覆盖主要场景）
7. **企业版/付费功能**：本次完全开源免费，后续可考虑企业定制服务

## Further Notes

### 隐私与合规

- 所有用户数据本地存储，不上传任何服务器
- 调用第三方AI前必须脱敏，脱敏映射表本地加密存储
- 爬取招聘平台时遵守robots.txt，控制请求频率，避免对平台造成压力
- 浏览器插件权限最小化，只请求必要的标签页访问权限

### 性能考量

- 本地AI模型首次下载体积大（4-8GB），需提供下载进度提示和取消功能
- 简历生成是CPU密集型任务，需在Rust后端异步执行，前端显示进度条
- SQLite数据库需建立合适的索引（岗位ID、简历ID、时间戳等）
- 爬取任务在后台线程执行，不阻塞前端UI

### 用户体验

- 首次使用提供引导流程（录入基本信息、配置爬取条件、选择AI模型）
- 空状态设计：无岗位时显示配置引导，无简历时显示创建引导
- 错误处理：网络失败、AI调用失败、爬取失败等场景提供友好的错误提示和重试机制
- 深色/浅色主题切换

### 社区运营

- 解析规则热更新仓库公开，欢迎社区贡献新平台适配
- 基础简历模板开源，鼓励用户分享自定义模板
- Issue模板和贡献指南需完善，降低参与门槛
